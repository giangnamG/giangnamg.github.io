## Overview

SMB cheatsheet theo 3 giai đoạn kiểm thử:

1. Chưa có user/password
2. Đã có user/password
3. Đã kết nối được vào SMB

Mục tiêu của bài này là tách rõ từng giai đoạn để biết chính xác ở mỗi thời điểm chúng ta nên làm gì, dùng công cụ nào, và khi nào nên dừng lại để chuyển sang bước tiếp theo.

## Default Ports

- `139/tcp`
- `445/tcp`

## Phase 1: Trinh sát tiền xác thực

Ý nghĩa: xác định dịch vụ SMB có đang mở không, target là máy gì, có dấu hiệu cấu hình yếu không, và có thể lấy được credential bằng password attack hay không.

### Mục tiêu

- Xác nhận `SMB` đang chạy trên `139/tcp` hoặc `445/tcp`
- Thu thập thông tin nền như hostname, domain/workgroup, phiên bản Windows, trạng thái `SMB signing`, `SMBv1`
- Kiểm tra xem có thể brute-force hoặc password spray trong phạm vi được phép hay không
- Chuẩn bị dữ liệu cho giai đoạn đăng nhập có xác thực

### Fingerprint SMB bằng NetExec

```bash
netexec smb <target-IP>
```

Ví dụ đầu ra thường cho biết:

- hostname
- domain hoặc workgroup
- phiên bản Windows / build
- `signing: True|False`
- `SMBv1: True|False`

### Service detection và banner grabbing bằng Nmap

```bash
# Quét nhanh 2 cổng SMB phổ biến
nmap -p 139,445 <target-IP>

# Lấy version / service banner
nmap -p 445 --open -sV <target-IP>

# Kiểm tra protocol SMB đang hỗ trợ
nmap --script smb-protocols -p 445 <target-IP>

# Kiểm tra security mode / signing
nmap -p 445 --script smb-security-mode <target-IP>
```

Những điểm nên đọc từ kết quả:

- target có mở `139` hay chỉ `445`
- version và khả năng hỗ trợ SMB
- `SMB signing` bật hay tắt
- giao thức cũ như `SMBv1` có còn bật không

### Password attack SMB bằng Hydra

```bash
hydra -L user.list -P password.list smb://<target-IP>
```

Lưu ý:

- Hydra thường tự giảm số luồng xuống thấp vì SMB không chịu được quá nhiều kết nối song song
- Nếu gặp lỗi `invalid reply from target`, nguyên nhân có thể là bản Hydra cũ không xử lý tốt `SMBv3`

### Password attack SMB bằng NetExec

NetExec có thể dùng trực tiếp cho SMB theo cú pháp chung:

```bash
netexec <proto> <target-IP> -u <user or userlist> -p <password or passwordlist>
```

Áp vào SMB:

```bash
netexec smb <target-IP> -u <user or userlist> -p <password or passwordlist>
```

Ví dụ:

```bash
netexec smb 10.129.42.197 -u user.list -p password.list
```

Khi thành công, NetExec sẽ trả về credential hợp lệ trên dịch vụ SMB của target.

### Password attack SMB bằng Metasploit khi Hydra lỗi

```bash
msfconsole -q
use auxiliary/scanner/smb/smb_login
set user_file user.list
set pass_file password.list
set rhosts <target-IP>
run
```

### Thử đăng nhập anonymous hoặc null session

Trước khi brute-force, nên kiểm tra nhanh xem SMB có cho truy cập `anonymous` hay không. Một số hệ thống cấu hình lỏng có thể cho liệt kê share hoặc truy cập dữ liệu mà không cần credential hợp lệ.

Có thể thử bằng `smbclient`:

```bash
smbclient -N -L \\\\<target-IP>\\
```

Ý nghĩa:

- `-N`: không dùng password
- `-L`: liệt kê share

Nếu thành công, đây là dấu hiệu rất tốt để enumerate tiếp mà chưa cần user/password.

Có thể kiểm tra thêm bằng `rpcclient` và `smbmap`:

```bash
# Null session bằng rpcclient
rpcclient -U "" <target-IP>

# Null session bằng smbmap
smbmap -H <target-IP> -u "" -p ""
```

### Enumeration không cần credential

Nếu target cho phép `anonymous/null session`, hoặc môi trường leak đủ thông tin qua SMB/RPC, có thể enumerate thêm user, group, domain và password policy ngay từ phase 1.

```bash
# Full enumeration
enum4linux -a <target-IP>

# Chỉ enumerate user
enum4linux -U <target-IP>

# Chỉ enumerate group
enum4linux -G <target-IP>

# Password policy
enum4linux -P <target-IP>
```

```bash
# Shares và users
nmap -p 445 --script=smb-enum-shares,smb-enum-users <target-IP>

# Groups và domains
nmap -p 445 --script=smb-enum-groups,smb-enum-domains <target-IP>
```

### Kết quả mong muốn của Phase 1

- Có được credential hợp lệ
- Hoặc xác định được target cho phép `anonymous/null session`
- Hoặc ít nhất có được fingerprint đủ tốt để ưu tiên target
- Nếu chưa có credential thì chưa đi tiếp sang phase 2

## Phase 2: Xác thực và lập bản đồ truy cập

Ý nghĩa: sau khi đã có `user/password`, việc quan trọng nhất không phải là vào share ngay, mà là xác định tài khoản đó nhìn thấy gì và được phép làm gì.

### Mục tiêu

- Xác thực credential có hợp lệ hay không
- Xác định tài khoản thuộc local machine hay domain
- Liệt kê các share khả dụng
- Xác định quyền trên từng share: `READ`, `WRITE`, hoặc không có quyền

### Kiểm tra credential và enumerate share bằng NetExec

```bash
netexec smb <target-IP> -u <username> -p <password> --shares
```

Ví dụ:

```bash
netexec smb 10.129.42.197 -u "user" -p "password" --shares
```

Kết quả cần đọc kỹ:

- `ADMIN$`
- `C$`
- `IPC$`
- các share business như `SHARENAME`
- cột `Permissions`

### Cách đọc nhanh kết quả

- `READ`: có thể duyệt và tải file
- `WRITE`: có thể upload, sửa, hoặc tạo file
- `ADMIN$`, `C$`: chỉ có giá trị nếu credential đủ mạnh, thường là local admin hoặc tương đương
- `IPC$`: hữu ích cho giao tiếp SMB/RPC, nhưng không phải share dữ liệu thông thường

### Xác định tài khoản thuộc local machine hay domain

Có thể xác định ngay từ ngữ cảnh xác thực của `NetExec`.

```bash
# Kiểm tra credential theo ngữ cảnh mặc định
netexec smb <target-IP> -u <username> -p <password>

# Ép xác thực bằng local SAM
netexec smb <target-IP> -u <username> -p <password> --local-auth

# Ép xác thực bằng domain
netexec smb <target-IP> -u <username> -p <password> -d <domain>
```

Cách đọc:

- Nếu output thành công theo dạng `HOSTNAME\user`, nhiều khả năng đây là ngữ cảnh local machine
- Nếu output thành công theo dạng `DOMAIN\user`, đây là domain account hoặc credential đang được xử lý trong ngữ cảnh domain
- Khi không chắc chắn, so sánh kết quả giữa `--local-auth` và `-d <domain>` để biết credential hợp lệ ở ngữ cảnh nào

### Liệt kê share và đọc quyền truy cập

Mục tiêu của bước này là trả lời 2 câu hỏi:

- target có những share nào?
- account hiện tại có quyền gì trên từng share?

#### Dùng NetExec để liệt kê share và đọc cột quyền

```bash
netexec smb <target-IP> -u <username> -p <password> --shares
```

NetExec phù hợp để đọc nhanh bức tranh tổng thể:

- tên share
- share hệ thống như `ADMIN$`, `C$`, `IPC$`
- cột `Permissions`

#### Dùng smbmap để xác nhận quyền thực tế

```bash
smbmap -H <target-IP> -u <username> -p <password>
```

`smbmap` phù hợp khi cần nhìn rõ account hiện tại có quyền gì trên từng share.

Cách đọc nhanh:

- `READ`: có thể duyệt và tải file
- `WRITE`: có thể upload, sửa, hoặc tạo file
- `NO ACCESS`: share nhìn thấy nhưng không dùng được

#### Khi nào dùng NetExec, khi nào dùng smbmap

- Dùng `NetExec` khi cần enumerate nhanh toàn bộ share và xem cột `Permissions`
- Dùng `smbmap` khi cần xác nhận rõ khả năng `READ/WRITE` trên từng share
- Thực tế thường dùng cả hai: `NetExec` để nhìn nhanh, `smbmap` để kiểm tra lại quyền

### Liệt kê hoặc kết nối vào một share cụ thể bằng smbclient

`smbclient` không phải công cụ chính để đọc ma trận quyền toàn hệ thống. Nó phù hợp hơn cho 2 việc:

- liệt kê share với credential
- kết nối trực tiếp vào một share cụ thể để thao tác

```bash
# Liệt kê share với credential
smbclient -L //<target-IP> -U <username>%<password>

# Kết nối trực tiếp vào share
smbclient //<target-IP>/<share> -U <username>%<password>
```

### Enumerate đệ quy nội dung share bằng smbmap

Sau khi đã xác định được share có quyền phù hợp, có thể dùng `smbmap` để enumerate sâu hơn nội dung bên trong share đó:

```bash
smbmap -H <target-IP> -u <username> -p <password> -r
```

### Checklist khi đã có credential

- Xác định rõ credential là local hay domain
- Xác định share nào có dữ liệu người dùng hoặc dữ liệu vận hành
- Xác định quyền thực tế trên từng share bằng `Permissions` hoặc `smbmap`
- Ưu tiên share có `READ,WRITE`
- Ghi chú rõ account đang dùng là account local hay domain
- Nếu nhiều credential hợp lệ, enumerate từng account riêng vì quyền SMB thường khác nhau

### Kết quả mong muốn của Phase 2

- Biết chính xác share nào truy cập được
- Biết quyền thực tế trên từng share
- Chọn được share mục tiêu để vào sâu ở phase 3

## Phase 3: Tương tác nội dung và thao tác trên share

Ý nghĩa: khi đã xác định được share có thể vào, trọng tâm chuyển sang duyệt nội dung, lấy file, đẩy file, và đánh giá khả năng ảnh hưởng thực tế.

### Mục tiêu

- Duyệt cấu trúc thư mục
- Liệt kê file
- Tải dữ liệu về để phân tích
- Upload file nếu có quyền ghi
- Xác định dữ liệu nhạy cảm, file cấu hình, script, backup, credential, hoặc cơ hội leo thang

### Kết nối bằng smbclient

```bash
smbclient -U <username> \\\\<target-IP>\\<share>
```

Ví dụ:

```bash
smbclient -U user \\\\10.129.42.197\\SHARENAME
```

Sau khi vào được shell `smb: \>`, có thể bắt đầu với:

```bash
ls
```

### Mount share trên Linux

Khi cần thao tác với share như một thư mục cục bộ, có thể mount bằng `CIFS`.

```bash
# Tạo mount point
sudo mkdir -p /mnt/smb_share

# Mount share bằng username/password
sudo mount -t cifs //<target-IP>/<share> /mnt/smb_share -o username=<username>,password=<password>
```

Nếu cần chỉ rõ domain:

```bash
sudo mount -t cifs //<target-IP>/<share> /mnt/smb_share -o username=<username>,password=<password>,domain=<domain>
```

Nếu muốn mount read-only để giảm rủi ro ghi nhầm:

```bash
sudo mount -t cifs //<target-IP>/<share> /mnt/smb_share -o username=<username>,password=<password>,ro
```

Khi dùng xong có thể unmount:

```bash
sudo umount /mnt/smb_share
```

### Những việc nên làm sau khi đã vào share

- Dùng `ls` để liệt kê file/thư mục
- Duyệt từng nhánh thư mục để hiểu cấu trúc lưu trữ
- Xác định file đáng chú ý như:
  - tài liệu nội bộ
  - script triển khai
  - file cấu hình
  - file backup
  - file chứa thông tin đăng nhập
  - khóa, chứng chỉ, hoặc dữ liệu người dùng
- Nếu có quyền ghi:
  - kiểm tra khả năng upload file
  - đánh giá rủi ro từ việc ghi đè nội dung hoặc cấy file mới

### Tải file và duyệt đệ quy

```bash
# Tải file riêng lẻ hoặc theo pattern trong smbclient
smbclient //<target-IP>/<share> -U <username>%<password>
> get sensitive_file.txt
> mget *.txt
```

```bash
# Tải đệ quy bằng smbget
smbget -R smb://<target-IP>/<share>/ -U <username>%<password>
```

```bash
# Liệt kê đệ quy bằng smbmap trên share cụ thể
smbmap -H <target-IP> -u <username> -p <password> -R <share>
```

### Tìm file mục tiêu

```bash
smbclient //<target-IP>/<share> -U <username>%<password>
> ls
> cd sensitive_folder
> get *.pdf
> get *.docx
```

### Câu hỏi cần trả lời ở phase này

- Có dữ liệu nhạy cảm nào đọc được không?
- Có đường dẫn nào dẫn đến credential reuse không?
- Có thư mục nào cho phép ghi trái phép không?
- Có cơ hội đặt file để hỗ trợ execution hoặc persistence không?
- Có share nào map tới thư mục làm việc của ứng dụng hoặc người dùng đặc quyền không?

### Kết quả mong muốn của Phase 3

- Có bằng chứng rõ ràng về dữ liệu truy cập được
- Có danh sách thư mục/file quan trọng
- Có đánh giá tác động thực tế của quyền `READ` hoặc `WRITE`

## Quick Workflow

```bash
# 1. Fingerprint SMB
netexec smb <target-IP>
nmap -p 139,445 <target-IP>
nmap -p 445 --open -sV <target-IP>
nmap --script smb-protocols,smb-security-mode -p 445 <target-IP>

# 2. Brute-force / password spray trong phạm vi cho phép
netexec smb <target-IP> -u <user or userlist> -p <password or passwordlist>
hydra -L user.list -P password.list smb://<target-IP>

# 3. Thử anonymous/null session
smbclient -N -L \\\\<target-IP>\\
rpcclient -U "" <target-IP>
smbmap -H <target-IP> -u "" -p ""
enum4linux -a <target-IP>
nmap -p 445 --script=smb-enum-shares,smb-enum-users,smb-enum-groups,smb-enum-domains <target-IP>

# 4. Nếu Hydra lỗi SMBv3, chuyển sang Metasploit
msfconsole -q
use auxiliary/scanner/smb/smb_login
set user_file user.list
set pass_file password.list
set rhosts <target-IP>
run

# 5. Có credential rồi thì enumerate share
netexec smb <target-IP> -u <username> -p <password>
netexec smb <target-IP> -u <username> -p <password> --local-auth
netexec smb <target-IP> -u <username> -p <password> -d <domain>
netexec smb <target-IP> -u <username> -p <password> --shares
smbclient -L //<target-IP> -U <username>%<password>
smbmap -H <target-IP> -u <username> -p <password>

# 6. Vào share mục tiêu
smbclient -U <username> \\\\<target-IP>\\<share>
```

## Operator Notes

- Không brute-force SMB ngoài phạm vi cho phép vì rất dễ tạo lockout hoặc log nổi bật
- Luôn đọc kỹ quyền `READ/WRITE`, đừng giả định rằng login thành công nghĩa là có quyền hữu ích
- `WRITE` trên share nội bộ thường có giá trị cao hơn việc chỉ enumerate bề mặt
- Nếu credential dùng được trên SMB, cần kiểm tra khả năng tái sử dụng trên các dịch vụ khác như `WinRM`, `RDP`, `SSH`
- Các kỹ thuật như exploit CVE, hash dumping, credential harvesting và persistence không nên đưa vào cheatsheet mặc định trừ khi có bối cảnh ủy quyền và mục tiêu kiểm thử rất rõ
