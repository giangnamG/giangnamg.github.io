# **Lab 03: Inconsistent security controls**

```jsx
┌────┬────────────────────────────────────────┬────────────────────────────────┐
│ #  │ Kiểm tra                               │ Lỗi nếu                        │
├────┼────────────────────────────────────────┼────────────────────────────────┤
│ 1  │ Register vs Change Email               │ Verification khác nhau        │
│ 2  │ Create Account vs Update Account       │ Validation khác nhau          │
│ 3  │ Web vs API vs Mobile                   │ Security controls khác nhau   │
│ 4  │ Admin create user vs User self-register│ Bypass verification           │
│ 5  │ Initial setup vs Subsequent changes    │ Thiếu re-verification         │
└────┴────────────────────────────────────────┴────────────────────────────────┘
```

Server **chỉ verify email domain khi đăng ký** (qua confirmation link), nhưng **KHÔNG verify lại khi user đổi email** sau đó. Attacker có thể đăng ký email bất kỳ → đổi thành email công ty → có quyền admin.


# **Lab 05: Low-level logic flaw**

Server **không kiểm tra integer overflow** khi tính tổng tiền. Khi giá trị vượt quá giới hạn của **signed 32-bit integer** (2,147,483,647), nó **wrap around thành số âm** (-2,147,483,648). Attacker có thể thêm sản phẩm đến khi tổng tiền "tràn" thành số âm hoặc gần 0.


# **Lab 12: Bypassing access controls using email address parsing discrepancies**

Server sử dụng **2 parser khác nhau** để xử lý email:

- **Validation parser**: Kiểm tra email có kết thúc bằng `@ginandjuice.shop` không
- **Email delivery parser**: Gửi email xác nhận đến địa chỉ

Hai parser này xử lý **encoded email khác nhau**. Bằng cách dùng **UTF-7 encoding**, attacker có thể khiến:

- Validation parser thấy: `...@ginandjuice.shop` → PASS ✓
- Email parser thấy: `attacker@exploit-server.net` → Gửi email đến đây

**Flow đăng ký bình thường:**

```jsx
┌─────────┐                              ┌─────────┐
│  User   │                              │ Server  │
└────┬────┘                              └────┬────┘
     │                                        │
     │  POST /register                        │
     │  email=user@ginandjuice.shop           │
     │───────────────────────────────────────>│
     │                                        │
     │                         ┌──────────────┴──────────────┐
     │                         │ Validation:                 │
     │                         │ email.endsWith(             │
     │                         │   '@ginandjuice.shop')?     │
     │                         │ → YES ✓                     │
     │                         │                             │
     │                         │ Send email to:              │
     │                         │ user@ginandjuice.shop       │
     │                         └──────────────┬──────────────┘
     │                                        │
     │  Email sent to: user@ginandjuice.shop  │
     │<───────────────────────────────────────│
     ▼                                        ▼
```
