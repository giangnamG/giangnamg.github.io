# **1. What is CSP (content security policy)?**

`CSP` là cơ chế bảo mật của trình duyệt nhằm mục đích giảm thiểu XSS và một số cuộc tấn công khác. Tính năng này hoạt động bằng cách hạn chế các tài nguyên (như `scripts` và `images`) mà một trang có thể tải và hạn chế việc một trang bên ngoài có thể load nội dung thông qua thẻ `framed`

# 2. Giảm thiểu các cuộc tấn công XSS bằng cách sử dụng CSP

Việc triển khai CSP được thực hiện thông qua tiêu đề phản hồi hoặc bằng cách kết hợp các phần tử meta vào trang HTML. Theo chính sách này, trình duyệt chủ động thực thi các quy định này và ngay lập tức chặn mọi vi phạm được phát hiện.

- Triển khai thông qua `response header`:
    
    ```scss
    Content-Security-policy: 
    default-src 'self'; img-src 'self' allowed-website.com; style-src 'self';
    ```
    
- Triển khai thông qua `meta tag`:
    
    ```scss
    <meta http-equiv="Content-Security-Policy" content="default-src 'self'; img-src https://*; child-src 'none';">
    ```
    

# 3. CSP **Defining Resources**

CSP hạn chế nguồn gốc để loading cả nội dung chủ động và thụ động, kiểm soát các khía cạnh như thực thi JavaScript nội tuyến và sử dụng `eval()`. Một chính sách ví dụ là:

```scss
default-src 'none';
img-src 'self';
script-src 'self' https://code.jquery.com;
style-src 'self';
report-uri /cspreport
font-src 'self' https://addons.cdn.mozilla.net;
frame-src 'self' https://ic.paypal.com https://paypal.com;
media-src https://videos.cdn.mozilla.net;
object-src 'none';
```

# 4. Chỉ thị - **Directives**

Danh sách tất cả các chỉ thị CSP phổ biến và ý nghĩa của chúng:

### 1. **default-src**

- **Mô tả**: Thiết lập nguồn mặc định cho tất cả các loại tài nguyên nếu không có các chỉ thị cụ thể khác. Nếu các chỉ thị như `script-src`, `style-src`, v.v., không được xác định, thì `default-src` sẽ áp dụng cho chúng.
- **Ví dụ**:
    
    ```
    default-src 'self';
    ```
    

### 2. **script-src**

- **Mô tả**: Chỉ định nguồn cho các tập lệnh JavaScript mà trang web có thể tải và thực thi.
    - Ví dụ: `script-src 'self'`
        - Chỉ cho phép **script (JavaScript)** được tải và thực thi từ chính domain của trang (`self`). Điều này sẽ **chặn** các tập lệnh được tải từ các nguồn bên ngoài hoặc các tập `lệnh **nội tuyến** (inline scripts)`.
        - **Quan trọng**: Vì chỉ cho phép script từ `self` và không có `unsafe-inline`, điều này **ngăn chặn** bất kỳ `mã JavaScript nội tuyến` nào, bao gồm các sự kiện như `onerror` được sử dụng trong thẻ HTML.
- **Ví dụ**:
    
    ```
    script-src 'self' https://apis.google.com;
    ```
    

### 3. **style-src**

- **Mô tả**: Chỉ định nguồn cho các tệp CSS mà trang web có thể tải.
- **Ví dụ**:
    
    ```
    style-src 'self' https://fonts.googleapis.com;
    ```
    

### 4. **img-src**

- **Mô tả**: Chỉ định nguồn cho các hình ảnh mà trang web có thể tải.
- **Ví dụ**:
    
    ```
    img-src 'self' https://images.example.com;
    ```
    

### 5. **font-src**

- **Mô tả**: Chỉ định nguồn cho các phông chữ mà trang web có thể tải.
- **Ví dụ**:
    
    ```
    font-src 'self' https://fonts.gstatic.com;
    ```
    

### 6. **connect-src**

- **Mô tả**: Chỉ định nguồn cho các yêu cầu kết nối, như `XMLHttpRequest`, `WebSocket`, `fetch`, v.v.
- **Ví dụ**:
    
    ```
    connect-src 'self' https://api.example.com;
    ```
    

### 7. **media-src**

- **Mô tả**: Chỉ định nguồn cho các tài nguyên phương tiện (audio, video).
- **Ví dụ**:
    
    ```
    media-src 'self' https://media.example.com;
    ```
    

### 8. **object-src**

- **Mô tả**: Chỉ định nguồn cho các đối tượng như `&lt;object&gt;`, `&lt;embed&gt;`, và `&lt;applet&gt;`.
- **Ví dụ**:
    
    ```
    object-src 'none';
    ```
    

### 9. **frame-src**

- **Mô tả**: Chỉ định nguồn cho các trang được nhúng bằng `&lt;frame&gt;` hoặc `&lt;iframe&gt;`.
- **Ví dụ**:
    
    ```
    frame-src 'self' https://trustedframe.example.com
    ```
    

### 10. **frame-ancestors**

- **Mô tả**: Xác định nguồn nào có thể nhúng trang của bạn thông qua `&lt;frame&gt;`, `&lt;iframe&gt;`, hoặc `&lt;object&gt;`.
- **Ví dụ**:
    
    ```
    frame-ancestors 'self' https://trustedparent.example.com;
    ```
    

### 11. **worker-src**

- **Mô tả**: Xác định nguồn mà các worker (như Web Workers, Service Workers) có thể tải từ.
- **Ví dụ**:
    
    ```
    worker-src 'self';
    ```
    

### 12. **child-src**

- **Mô tả**: Chỉ định nguồn cho các nội dung được tải bởi các frame, worker, hoặc embedded content.
- **Ví dụ**:
    
    ```
    child-src 'self' https://example.com;
    ```
    

### 13. **manifest-src**

- **Mô tả**: Xác định nguồn cho các tệp manifest (JSON).
- **Ví dụ**:
    
    ```
    manifest-src 'self';
    ```
    

### 14. **form-action**

- **Mô tả**: Xác định các URL mà trang web có thể gửi form (qua `&lt;form action&gt;`).
- **Ví dụ**:
    
    ```
    form-action 'self' https://secure.example.com;
    ```
    

### 15. **base-uri**

- **Mô tả**: Chỉ định nguồn cho thẻ `&lt;base&gt;` (để thiết lập URL cơ sở cho tất cả các liên kết tương đối trong tài liệu).
- **Ví dụ**:
    
    ```
    base-uri 'self'
    ```
    

### 16. **plugin-types**

- **Mô tả**: Chỉ cho phép các loại MIME được tải dưới dạng plugin (như Flash, PDF).
- **Ví dụ**:
    
    ```
    plugin-types application/pdf;
    ```
    

### 17. **sandbox**

- **Mô tả**: Áp dụng các hạn chế cho các tài liệu được nhúng hoặc được thực thi. Có thể được sử dụng với các tùy chọn như `allow-scripts`, `allow-same-origin`, v.v.
- **Ví dụ**:
    
    ```
    sandbox allow-scripts;
    ```
    

### 18. **report-uri** (deprecated) và **report-to**

- **Mô tả**: Xác định URL để gửi báo cáo vi phạm CSP. `report-uri` đã bị thay thế bằng `report-to`.
- **Ví dụ**:
    
    ```
    report-to https://report.example.com;
    ```
    

### 19. **upgrade-insecure-requests**

- **Mô tả**: Yêu cầu tất cả các yêu cầu HTTP không an toàn phải được nâng cấp lên HTTPS.
- **Ví dụ**:
    
    ```
    upgrade-insecure-requests;
    ```
    

### 20. **block-all-mixed-content**

- **Mô tả**: Chặn mọi yêu cầu tải nội dung không an toàn (HTTP) khi trang được phục vụ qua HTTPS.
- **Ví dụ**:
    
    ```
    block-all-mixed-content;
    ```
    

### 21. **trusted-types**

- **Mô tả**: Giới hạn việc sử dụng DOM XSS bằng cách yêu cầu các script phải sử dụng **Trusted Types** để thao tác với DOM.
- **Ví dụ**:
    
    ```
    trusted-types default;
    ```
    

### 22. **require-trusted-types-for**

- **Mô tả**: Yêu cầu Trusted Types cho các thuộc tính hoặc phương thức nhất định (thường là cho `script` để ngăn chặn XSS).
- **Ví dụ**:
    
    ```
    require-trusted-types-for 'script';
    ```
    

### Các nguồn đặc biệt:

- **`'self'`**: Chỉ cho phép tài nguyên từ cùng một domain của trang web.
- **`'none'`**: Không cho phép bất kỳ tài nguyên nào.
- **`'unsafe-inline'`**: Cho phép các tập lệnh nội tuyến và CSS (không được khuyến khích do nguy cơ XSS).
- **`'unsafe-eval'`**: Cho phép sử dụng `eval()` trong JavaScript (không được khuyến khích).
- **`'strict-dynamic'`**: Cho phép các tập lệnh tự động thêm các tập lệnh động khác (thường dùng với CSP nâng cao).
- **`nonce-*`**: Cho phép tập lệnh với thuộc tính nonce (một giá trị ngẫu nhiên được thêm vào các thẻ `&lt;script&gt;` để cho phép chúng chạy).
- **`hash-*`**: Cho phép tập lệnh hoặc kiểu với giá trị băm (hash) cụ thể.

### Ví dụ về CSP đầy đủ:

```sass
Content-Security-Policy:
    default-src 'self';
    script-src 'self' https://apis.example.com;
    style-src 'self' https://fonts.googleapis.com;
    img-src 'self' data:;
    connect-src 'self' https://api.example.com;
    font-src 'self' https://fonts.gstatic.com;
    frame-ancestors 'none';
    form-action 'self';
    upgrade-insecure-requests;
    block-all-mixed-content;

```
